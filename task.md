# RootSpread 任务说明

## 项目概述
- 项目名称：根系蔓延（RootSpread）
- 产品定位：基于思维导图的项目管理工具
- 核心理念：项目即任务，任务即项目；每个节点都可以无限扩展出子任务
- 设计风格：Linear 风格，深色大底、科技感、克制动效、精密细节

## 已确认的产品决策
- 使用 Next.js + React 作为前端技术栈
- 使用 FastAPI 作为后端技术栈
- 使用 MySQL 作为主数据库
- 使用 Redis 作为缓存
- 邮件服务使用 Resend
- 主工作台支持思维导图视图与表格视图切换
- 主工作台支持切换里程碑
- 支持同级任务排序
- 不做跨父节点的树结构移动
- 待验证任务退回进行中时，备注为选填
- 统计功能先预留入口，后续再补完整能力

## 功能范围

### 1. 账号体系
- 邮箱注册
- 邮箱验证
- 登录/退出登录
- 注册后创建工作空间（团队）
- 创建者默认为空间管理员
- 空间管理员可邀请成员加入工作空间
- 成员管理与角色管理

### 2. 任务管理
- 默认视图为思维导图
- 支持切换到表格视图
- 每个节点包含以下字段：
  - 名称
  - 内容（Markdown）
  - 创建人
  - 负责人
  - 创建时间（自动填充）
  - 更新时间（自动填充）
  - 计划截止时间
  - 完成时间
  - 权重
  - 评分
  - 状态（进行中、待验证、已完成、终止）
- 支持状态筛选，可多选状态
- 支持同级排序
- 支持新增子任务、编辑节点、展开/收起节点

### 3. 任务状态规则
- 下级任务全部完成后，上级任务自动标记为已完成
- 有下级任务的上级任务不能手动标记为已完成
- 已完成任务新增下级任务后，自动改为进行中
- 待验证任务可被改为已完成或退回进行中
- 退回进行中时可填写备注，备注非必填
- 评分由空间管理员对成员任务完成情况进行打分

### 4. 里程碑
- 里程碑包含名称和时间节点
- 创建里程碑后，自动归档该时间点之前已完成、终止的任务
- 当前任务视图默认不显示归档任务
- 可切换里程碑查看历史任务
- 主工作台中可直接切换里程碑

### 5. 统计功能
- 先预留统计入口
- 后续再补充具体统计项

## 推荐技术方案
- 前端：Next.js App Router + React + TypeScript
- UI：Tailwind CSS + shadcn/ui + 自定义 Linear 风格设计 token
- 导图核心：@xyflow/react（React Flow）+ 自定义脑图布局
- 后端：FastAPI + SQLAlchemy 2.x + Alembic
- 数据库：MySQL
- 缓存：Redis
- 邮件：Resend
- 鉴权：JWT Access Token + Refresh Token
- 权限：基于 Workspace 的角色控制（owner/admin/member）

## 核心数据模型
- users
- workspaces
- workspace_members
- workspace_invitations
- email_verification_tokens
- refresh_tokens
- task_nodes
- task_status_transitions
- milestones
- milestone_snapshots

## 当前非目标
- 不支持跨父节点移动任务
- 不优先实现复杂实时协作
- 不优先实现完整统计分析
- 不优先实现高级思维导图编辑能力（如自由拖拽重构整棵树）

## 备注
- RootSpread 是项目管理工具，不是传统思维导图编辑器
- 导图能力服务于任务管理，而不是追求完整导图工具能力
- 所有设计与交互优先围绕项目管理场景收敛
- 当前主工作台已升级为 React Flow 思维导图画布，支持节点聚焦、历史里程碑只读查看、画布缩放拖拽与子任务快速创建
- 当前已补齐节点详情侧栏、任务删除、状态流转记录、成员邀请与角色调整，后续继续补强邀请状态与更多节点快捷操作
- 当前里程碑面板已支持创建说明、时间点归档、历史切换与快照查看；邀请面板已支持邀请记录展示、撤销邀请、复制调试 token 与接受链接
- 当前已补齐批量任务操作、节点快捷状态菜单、同级拖拽排序反馈、统计页与审计日志接口
- 当前已建立前后端基础自动化测试：后端使用 Pytest 覆盖账号、工作空间、任务、里程碑、统计与审计；前端使用 Vitest + Testing Library 覆盖成员面板、节点详情侧栏和统计页关键渲染逻辑
